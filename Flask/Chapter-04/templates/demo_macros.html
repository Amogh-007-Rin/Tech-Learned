Jinja2 Macros Demo

This page demonstrates the use of Jinja2 macros for reusable template components.

Macro Definitions

Macros are defined in the macros.html file and imported here:

{% from 'macros.html' import user_card, alert_box, button_group %}
Available Macros:
  • user_card(user, show_actions=True) - Display user information
  • alert_box(message, type='info') - Display alert messages
  • button_group(buttons) - Display button groups
User Cards (Using Macros)
{% for user in users %}
{{ user_card(user, show_actions=true) }}
{% endfor %}
Alert Boxes (Using Macros)
{{ alert_box('This is a success message!', 'success') }} {{ alert_box('This is an info message!', 'info') }} {{ alert_box('This is a warning message!', 'warning') }} {{ alert_box('This is an error message!', 'danger') }}
Button Groups (Using Macros)
Action Buttons:
{{ button_group([ {'text': 'Edit', 'class': 'btn-primary', 'onclick': 'alert("Edit clicked")'}, {'text': 'Delete', 'class': 'btn-danger', 'onclick': 'alert("Delete clicked")'}, {'text': 'View', 'class': 'btn-info', 'onclick': 'alert("View clicked")'} ]) }}
Navigation Buttons:
{{ button_group([ {'text': 'Previous', 'class': 'btn-outline-secondary', 'onclick': 'alert("Previous clicked")'}, {'text': 'Next', 'class': 'btn-outline-primary', 'onclick': 'alert("Next clicked")'} ]) }}

Definition:

{% endmacro %}
Import:

Usage:
{{ macro_name(value1, value2) }}
Example:
{% macro user_card(user) %}
    

        

            
{{ user.name }}

{% endmacro %}
Macro vs Include
Macros:
  • • Reusable functions
  • • Accept parameters
  • • Return HTML
  • • Defined once, used many
Includes:
  • • Static content
  • • No parameters
  • • Complete templates
  • • Copied as-is
Best Practices
  • • Keep macros simple
  • • Use descriptive names
  • • Provide default values
  • • Document parameters
  • • Group related macros
